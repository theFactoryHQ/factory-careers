# Production PDF Parser Recovery Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Restore reliable PDF resume parsing in production, prevent worker-bundle regressions, and backfill every currently affected candidate document.

**Architecture:** Keep `pdf-parse` as the single PDF text-extraction implementation. Trace PDF.js's dynamically loaded worker into Nitro, then make the post-build gate run a real text extraction through the bundled package instead of checking only that a file exists. Validate against generated non-PII input and the production corpus, then backfill null `parsed_content` records only after deployment.

**Tech Stack:** Nuxt 4, Nitro, TypeScript, Node.js, `pdf-parse`/PDF.js, Vitest, PostgreSQL, S3-compatible object storage, Render.
